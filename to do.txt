تنفيذ مصادقة JWT كاملة: تسجيل/تفعيل بريد إلكتروني/تسجيل دخول/refresh tokens
Middleware لاستخراج tenant_id من الـ JWT وحقنه في سياق الطلب
إضافة Category (model + migrations + serializers + services + views + urls + tests) — مقيدة بالـ store
إضافة Product مع ProductImage وInventory (model + migrations + serializers + services + views + urls + tests) — مقيدة بالـ store
دعم رفع الصور وتخزينها (محلي أولاً) وربطها بـ ProductImage
فرض تحقق بيانات المنتج (SKU فريد per store، حقول إلزامية، قيود السعر/الكمية)
نقاط نهاية API كاملة لإدارة Categories وProducts مع فحوصات الملكية/الأذونات
تحسين قاعدة البيانات: إضافة فهارس على tenant_id وsku + إنشاء migrations
ضمان تطبيق التصفية المقيّدة بالـ tenant في كل selectors/repositories + كتابة اختبارات تحقق تلقائية
آلية توليد subdomain وواجهة اقتراح/التحقق من اسم المتجر الفريد
اختبارات تكامل: عزل المستأجر (two stores/users) على Categories/Products، واختبارات رفع الصور وتحديث المخزون
توثيق الـ API (OpenAPI / drf-spectacular) للواجهات الجديدة
إضافة تسجيل (logging) ومعالجة أخطاء منسقة في خدمات المنتجات والتصنيفات
CI بسيط: تشغيل اختبارات الباك‑إند على كل دفع (GitHub Actions)
